What was done today
-------------------
1) Project reset + architecture simplification
- Large clean-slate pass removed old auth flows, legacy UI kit/gluestack wrappers, old hooks/contexts, old catalog/product/design abstractions, and legacy scripts.
- Tailwind/NativeWind and related config files were removed from the app baseline.
- Many legacy image assets and older camera/web artifacts were removed.

2) App structure moved toward a minimal native-tabs baseline
- Root routing is now centered on Expo Router with `(tabs)` as the default entry.
- `app/index.tsx` now redirects to the tabs group.
- `app/(tabs)/_layout.tsx` uses `expo-router/unstable-native-tabs`.
- Tab routes currently in place: `index`, `camera`, `marketplace`, `account`.
- Placeholder screens were added/updated for camera, marketplace, and account to establish clean scaffolding for future implementation.
- Root layout (`app/_layout.tsx`) includes iOS `@expo/ui/swift-ui` Host wrapping with a React Native fallback.

3) Backend narrowed to Firebase-first strategy
- README and backend docs now state Stripe should be handled by the Firebase Stripe extension (extension-only direction).
- Custom Firebase Functions were trimmed to non-Stripe utilities.
- `functions/src/index.ts` now includes:
  - `healthCheck` endpoint
  - `generateGarmentQRCodes` endpoint (IAM private invoker)
- QR generation flow now:
  - reads Firestore `Garments`
  - deduplicates garment IDs
  - skips already generated entries
- writes PNG QR codes to Storage under `Garments/<garmentId>/code_1200x1200.png`
  - sets `garmentQRCodeStatus: "Generated"` on garment docs
- `qrcode` dependency added in `functions/package.json`.
- Firestore rules tightened to backend-only baseline (`allow read, write: if false;`).

4) Documentation and setup updates
- `README.md` rewritten to describe clean-slate state and Firebase backend direction.
- `functions/README.md` updated with function behavior, IAM security notes, local usage, and deploy commands.
- New planning/reference docs were added under `documentation/` (DB design + design philosophy rules).
- `eas.json` is present for EAS workflow setup.
